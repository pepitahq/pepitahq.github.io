Developer tools

Drive your website from Claude, or the terminal.

pepita is a website builder for people who'd rather not build a website. These two tools do the same things the editor does — read files, make edits, publish, and spin up shareable preview links — without opening it. Same account, same sites, no new passwords.

claude — pepita connector
List my pepita sites, then read index.html on my-landing.
→ 3 sites. my-landing/index.html (52 KB), styles.css, …
Change the hero headline to "Roofing done right".
→ wrote index.html
Give me a preview link I can send my client.
→ preview ready → my-landing--a1b2.pepita.page
Looks good — publish it.
→ published → my-landing.pepita.page
MCP

The Claude connector

Connect pepita to Claude (Claude Code, Claude Desktop, or the MCP Inspector) and edit your sites in plain language. It exposes ten tools — five that read your sites, five that edit them, publish, or manage shareable preview links. Edits show up live in the pepita web editor too, so you can move between the chat and the visual editor mid-task.

Tools

list_sitesYour sites and their URLs.read
get_statusA site's pending changes and links.read
list_site_filesThe file tree of a site.read
read_site_fileThe contents of one file.read
write_site_fileCreate or edit a file in the site.writes
publishPut the current site live.writes
create_previewFreeze the current site as a new shareable link.writes
update_previewPush the current site onto an existing preview link.writes
list_previewsA site's active preview links.read
delete_previewStop serving a preview link (restorable from History).writes
What it can't do, on purpose. Custom domains, team management, billing, and deleting a site stay in the editor — where their confirmations live. The connector can only do what you can do, on your own sites.

Connect it

local — stdio
# 1. authorize this device once (opens the browser)
npx @pepitahq/cli login
# 2. add the connector to Claude Code
claude mcp add pepita -- npx -y @pepitahq/mcp

A remote connector (OAuth sign-in, nothing to install) is on the way to the Claude Connectors directory. The local server above works today with any MCP client.

CLI

The command line

@pepitahq/cli pulls a site to your machine, lets you edit it with local tools, and pushes it back — authenticated the way wrangler or gh do it: a browser sign-in, no pasted API key.

Install & sign in

terminal
npm install -g @pepitahq/cli
pepita login
→ authorized as you@example.com

Commands

pepita listYour sites.
pepita create <name>Create a new site (optionally from a local folder).
pepita pull <slug>Download a site's files to a local folder.
pepita apply <slug>Upload local changes into the working copy (confirm-gated).
pepita preview <slug>Create, update, or remove a shareable preview link.
pepita previews <slug>List active preview links.
pepita publish <slug>Put the current site live.
pepita status <slug>Pending changes and URLs.
pepita delete <slug>Permanently delete a site (optional snapshot first).
pepita whoami · logoutWho you are · sign out (revokes the device).

A round trip

terminal
pepita pull my-landing --dir ./my-landing
# …edit files with your own tools…
pepita apply my-landing --dir ./my-landing
pepita preview my-landing # a shareable link to review first
pepita publish my-landing
→ published → my-landing.pepita.page
Revoke anytime. Every device you authorize appears in the editor under Settings → Connected devices, where you can sign it out.
